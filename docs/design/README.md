# Design documents

redraft's detailed design documents — the canonical-store and index design, the
MCP server design, the operator-UI design, and the report-bundle schema — are
internal working documents. They are referenced throughout the source by name and
section (e.g. `s6-ui.md §9.1`, `design-storage.md §7.3`) as design provenance, but
they are kept in the project's private development history rather than published
here.

The code itself and the organizing protocol (`docs/protocol/organizing-protocol.md`)
are the public source of truth for how redraft works.
